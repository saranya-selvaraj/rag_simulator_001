RAG Simulator

Interactive simulation of Retrieval-Augmented Generation (RAG). This demonstrates how documents are chunked, embedded into vectors, retrieved with cosine similarity, and finally used by an LLM to answer questions.

📚 Knowledge Base Documents

❓ Ask a Question

1

Indexing

Split documents into chunks and convert each chunk into embeddings (vectors).

2

Query Embedding

Convert the user's question into a vector using the same embedding process.

3

Retrieval

Compare vectors using cosine similarity and retrieve top matching chunks.

4

Generation

The LLM receives the retrieved chunks as context and generates an answer.
